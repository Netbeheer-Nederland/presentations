== Wat is LinkML?

LinkML is een *taal* voor het modelleren van:
* data/informatie
* vocabulaire

== Single source of truth

Gangbaar:
* databaseschema
* in-memory model in programmeertaal
* schema voor berichtuitwisseling
* visualisatie

* ieder van deze:
* technische details voor nodig
* meerdere plekken van onderhoud gelijktijdig

== Maar met LinkML...

* single source of truth schema
* genereren van overige modellen en artefacten
* visualisatie ook cadeau
* focus op domein want
** technologie-agnostisch
** simpel (YAML + OO)

== Eerlijke kritische blik

Wanneer codegeneratie niet per se handiger is:

* hoge mate van technische commitment (afnemer is SQL)
* hoge performancevereisten

== Semantiek

* Voor mensen
** Eenvoudig duiden dat termen uit standaarden lenen
** Ontkoppeling eigen taal en dat van standaard gerelateerd via mappingmetadata

* Voor machines
** URI's als globale identifier: machine snapt identiteit over de hele wereld
** Knowledge graph enablen
*** Context-loze feiten opslaan


== DevOps werkwijze mogelijk




[unstyled%step]
* Technology agnostic with focus on modeling of the domain/application
* Logical schema as single source of maintenance
* Automatic deterministic generation of other schemas and documentation
* Machine-processable and human friendly YAML syntax
* Expressive but familiar modeling OO inspired constructs
* Emphasis on semantics
* Lots of standardized metadata support
* FAIR and native Linked Data support
